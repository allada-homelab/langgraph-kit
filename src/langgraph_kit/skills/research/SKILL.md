# Research Skill

## Workflow

1. **Define question**: Clearly state the research question or objective before starting.
2. **Plan sources**: Identify which sources to check — codebase, documentation, web, memory.
3. **Gather evidence**: Search each source systematically. Record findings with source references.
4. **Synthesize**: Combine findings into a coherent answer. Note conflicts between sources.
5. **Report**: Deliver findings in the structured format below.

## Research Principles

- **Breadth before depth**: Survey all relevant areas before deep-diving into one.
- **Track provenance**: Every claim must have a source (file path, URL, memory reference).
- **Note uncertainty**: If information is partial or conflicting, say so explicitly.
- **Limit scope**: Stay within the defined research question. Flag tangential discoveries for later.

## Output Format

```
## Research: [question]

### Findings
1. **[Topic]** — [finding] (source: [reference])
2. **[Topic]** — [finding] (source: [reference])

### Conflicts / Uncertainty
- [description of conflicting information]

### Conclusion
[1-3 sentence synthesis answering the original question]

### Further Investigation
- [optional: topics that surfaced but were out of scope]
```
